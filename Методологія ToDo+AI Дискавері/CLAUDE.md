# Контекст папки: Методологія ToDo+AI Дискавері

Авторське дерево методології Odoo 19 (етапи ІНІЦІАЦІЯ і ДИСКАВЕРІ) — скіли та інструкції, розкладені по блоках процесу.

## Структура

- `Методологія ToDo+AI/Скіли для Дискавері + MVP · Odoo 19/1. ІНІЦІАЦІЯ/` — 12 скілів по блоках:
  `1. Аналіз інформації+ дослідження клієнта` (5), `2. Підготовка до Установчої Зустрічі` (4),
  `3. Аналіз Установчої Зустрічі + план дискавері` (3); у кожному блоці — `ІНСТРУКЦІЯ.md`,
  у корені дерева — майстер-`ІНСТРУКЦІЯ.md`.
- `Методологія ToDo+AI/Скіли для Дискавері + MVP · Odoo 19/2. ДИСКАВЕРІ/` — етап циклу зустрічей:
  майстер-`ІНСТРУКЦІЯ.md` у корені; блок `1. Підготовка питань` (2 скіли: `session-question-builder`
  з контрактом журналу циклу, `session-client-prep-builder` з бланком листа; `ІНСТРУКЦІЯ.md`,
  `ТЕСТ_циклу_3-ітерації.md`). Наступні блоки етапу (аналіз транскрипту сесії, маппінг scope) — у розробці.
- `Методологія ToDo+AI/bpmn_odoo19_discovery.html` — копія канонічної BPMN
  (канон — `bpmn_odoo19_discovery_СТАТУС-скілів.html` у сусідній папці «Методологія ToDo+AI»).
- `Архів/`, `ПЛАН_*`, `РЕВ'Ю_*`, `валідатор_скілів.py` — історичні матеріали перевірок.

## Правила

1. **Джерела істини пакетів** — сусідня папка «Методологія ToDo+AI»: `СКІЛИ_виправлені/` (ІНІЦІАЦІЯ)
   і `СКІЛИ_ДИСКАВЕРІ/` (ДИСКАВЕРІ) — звідти збираються й інсталюються `.plugin`.
   Це дерево — авторська копія для читання по блоках.
2. При зміні скілів синхронізувати обидві копії свідомо (відомий випадок розбіжності — `industry-briefing`,
   де новіша редакція була саме тут; з v0.5.4 копії вирівняні).
3. Інсталяція плагінів: `.plugin` доставляти через папку outputs сесії Cowork — лише так з'являється кнопка встановлення.

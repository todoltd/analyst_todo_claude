# odoo19-discovery-closeout

Методологія блоку **ЗАВЕРШЕННЯ ДИСКАВЕРІ** для проєктів впровадження Odoo 19 — вихід із циклу
зустрічей (`odoo19-discovery-sessions`) у фазу ТР і білд (`odoo19-mvp-build`).

Вмикається, коли всі теми плану в `журнал_дискавері` закриті (✅). Консолідує журнал + протоколи
у deliverables дискавері і будує міст у ТР.

## Скіли (5, вузька нарізка — один скіл = одна дія)

1. `discovery-scope-mapper` — гейт готовності циклу + консолідація scope: що в scope
   (модулі/процеси/інтеграції з джерелами), що поза scope (виключення X-N), кандидати на кастомізацію.
   Результат — `scope_[компанія].md`.
2. `discovery-gap-analyzer` — fit-gap: AS-IS → TO-BE (стандартний Odoo 19, зі звіркою з докою) →
   gap-list → customization-list. Результат — `gap-аналіз_[компанія].md`.
3. `discovery-report-builder` — підсумковий звіт замовнику (Humizer + фільтр чутливого), `.md` +
   брендований `.docx` (бланк ToDo вшито). Результат — `discovery-звіт_[компанія].md` + `.docx`.
4. `journal-to-tr-builder` — **міст у ТР**: скаффолд `ТР_[назва].md` + секція 1 «Функціональні вимоги»
   з `origin` (D-N/T-N/сесія N) + плейсхолдер під `tr-usecases-acceptance`. Закриває розрив «цикл → ТР».
5. `discovery-sow-builder` — комерційна пропозиція / SoW (обсяг із customization-list, орієнтовна
   оцінка із застереженням «уточнюється в ТР»), `.md` + брендований `.docx`. Результат — `discovery-КП_[компанія].md` + `.docx`.

## Місце в процесі

```
ДИСКАВЕРІ цикл (усі теми ✅)
    ↓
discovery-scope-mapper   → scope_[компанія].md
    ↓
discovery-gap-analyzer   → gap-аналіз_[компанія].md (AS-IS → TO-BE → gap-list)
    ↓
discovery-report-builder → discovery-звіт_[компанія].md + .docx (замовнику)
    ↓
journal-to-tr-builder    → ТР_[назва].md (секція 1 + origin) → ланцюжок tr-* → MVP/білд
```

## Збірка та інсталяція

```bash
cd "СКІЛИ_ЗАВЕРШЕННЯ" && bash ЗІБРАТИ_plugin.sh
```

Інсталяційна картка з кнопкою з'являється лише коли `.plugin` доставлено через папку **outputs** сесії Cowork.

## QA

`валідатор_скілів_дискавері.py` (у сусідньому пакеті `СКІЛИ_ДИСКАВЕРІ/`) перевіряє всі пакети методології —
інваріанти вмісту, заборонені терміни, баланс код-фенсів, крос-посилання:
`python3 валідатор_скілів_дискавері.py "../СКІЛИ_ЗАВЕРШЕННЯ/skills" "skills"`.

## Джерело істини

scope / gap / звіт / ТР — проекції з `журнал_дискавері` (єдине джерело стану циклу). Вимоги в ТР несуть
`origin:` (D-N / T-N / сесія N) — наскрізна трасованість до коду й тестів (контракт SDD-хребта,
`КОНТРАКТ_SDD-хребет_статуси-і-ID.md`).

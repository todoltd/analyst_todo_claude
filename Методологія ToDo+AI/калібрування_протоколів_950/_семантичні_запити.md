Запускаються через `search_knowledge_base` БЕЗ фільтра проєкту (вся база) — дають сигнал поза 180 глибокими.
Колонка «крит.» = який критерій ідеалу живить (1 менше ручних правок / 2 структура / 3 Humizer / 4 повнота).
`collections`: D = documents (протоколи), T = transcriptions (сирі транскрипти — джерело «було» для Humizer).

## Метод ЕТАПу 2 (нагадування собі)
- 180 глибоких — батчами по ~25–30 через субагентів (читають повний текст батчу, повертають структуровані спостереження; raw-текст НЕ заходить у головний контекст).
- Кожен батч → append у `_патерни_лог.md` (структура / канонічні формулювання / анти-формулювання / ручні доправлення / варіативність за типом).
- 26 семантичних запитів — окремою секцією «семантичні сигнали по всій базі» в тому ж лозі.
- Проміжний Чекпойнт 2 після ~60–80 опрацьованих.

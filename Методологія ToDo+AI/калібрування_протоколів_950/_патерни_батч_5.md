## Структурні факти

### Типовий порядок секцій (standard канонічний):
```
Шапка (дата, місце, тема, мета, документ створив, учасники)
→ Питання для обговорення (таблиця №/Питання/Обговорювали)
→ РЕЗУЛЬТАТИ ІНТЕРВ'ЮУ (підрозділи)
→ ВИРІШИЛИ (таблиця №/Рішення/Термін/Відповідальний)
→ ДОДАТОК 1 (шаблон або заповнений)
```

### Варіанти РЕЗУЛЬТАТІВ (спостережені в батчі):
1. **Класичний (Функц./Нефункц.)** — RSE CRM, RSE HR, ELKO дизайн, Peran Food — розділ Функціональні вимоги + Нефункціональні
2. **Блоковий (з підзаголовками)** — Агро-Технік, Хайтек — кожен блок: Контекст + Підрозділи + Функц. вимоги + Нефункц. вимоги + Прийняті рішення + Технічні деталі + Наступні кроки
3. **Q&A нумерований** — Yodezeen Фінанси (1.1, 1.2…)
4. **Вільний текст** — UTTC Склад, Variant Agro, Get-IT CRM
5. **Процесний (As-Is/To-Be)** — SolarSK, Трител ПДВ, Хайтек (два додатки)
6. **Аналітичний (нова структура)** — Ельфа-Лабораторія «1. АНАЛІЗ ЗУСТРІЧІ — ЯК Є»

### Секції «завжди присутні»:
- Шапка (100%)
- ВИРІШИЛИ або аналог (27/30; відсутні в #1 ICONIC, #20 UTTC; #4 Ельфа замінено таблицею питань)

### Секції «опціональні»:
- Відкриті питання (≈15/30)
- Перелік файлів / Отримані від Замовника (≈12/30)
- Пропозиції учасників (≈10/30)
- ДОДАТОК 1 незаповнений шаблон (≈20/30)
- ДОДАТОК 1 заповнений (3/30: SolarSK, Хайтек, Агро-Технік)
- Наступні кроки в тілі (≈6/30)

### Відхилення від канону:

## Варіативність за типом (з батчу)

### Установча (2/30: Variant Agro, Смарт Блок)

### Збір вимог (≈15/30)
- Найширша варіативність структури РЕЗУЛЬТАТІВ

- Старі протоколи (2023–2024) = вільний текст або Q&A
- Відкриті питання присутні у ≈70% збір-вимог протоколів

# Блок «Аналіз сесії» — інструкція

**Місце в процесі:** після кожної зустрічі циклу дискавері, перед протоколом
(вузли BPMN: «ЗУСТРІЧ» → «Аналіз транскрипту (Claude)» → «Протокол»).

## Скіл блоку

### session-transcript-analyzer — аналіз транскрипту сесії + журнал (головний writer журналу)

**Коли:** одразу після зустрічі, коли є транскрипт / нотатки / заповнений опитувальник.
**Тригер:** "проаналізуй сесію [N] для [компанія]" (також: "розбери транскрипт сесії", "онови журнал з сесії").
**Вхід:** транскрипт/нотатки, заповнений `питання_сесії-[N]`, `журнал_дискавері`, план дискавері.
**Вихід:** оновлений `журнал_дискавері_[компанія].md` + короткий `аналіз_сесії-[N]_[компанія].md`.

Що робить за тебе:
- розкладає сказане по реєстрах журналу: рішення (D-N), виключення зі scope (X-N), перенесення (C-N),
  нові теми (T-N), відкриті питання (Q-N), обіцяні артефакти виконавця (A-N);
- проставляє статуси тем плану (✅ / 🔶 / ⬜), оновлює шапку журналу (дата + враховані джерела);
- продовжує наскрізні ID (не перенумеровує), переживає ручні правки БА;
- сигналить «цикл можна завершувати», коли всі теми ✅ → перехід у блок «Завершення».

**Розмежування:** аналіз — внутрішній стан + вхід у протокол; протокол замовнику робить
`meeting-protocol-builder` (ІНІЦІАЦІЯ); НЕ для Установчої Зустрічі (для неї — `kickoff-transcript-analyzer`).

**Наступний крок:** `meeting-protocol-builder` (протокол замовнику); якщо лишились теми —
`session-question-builder` для наступної сесії; якщо всі теми ✅ — `discovery-scope-mapper` (блок «Завершення»).

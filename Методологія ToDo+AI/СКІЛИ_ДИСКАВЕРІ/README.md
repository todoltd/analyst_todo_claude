# odoo19-discovery-sessions

Методологія етапу **ДИСКАВЕРІ — цикл зустрічей** для проєктів впровадження Odoo 19.
Працює в парі з плагіном `odoo19-discovery-initiation` (його фінал — `план_дискавері` — це вхід циклу).

## Скіли

### Блок «Підготовка питань»

1. `session-question-builder` — внутрішній опитувальник до КОЖНОЇ сесії циклу з пам'яттю контексту:
   що вже обговорено (не перепитує), що перенесено з минулої зустрічі (ставить першим),
   що виключено зі scope (не питає). Критичні питання — з навідними підказками для неговірких
   замовників. Веде `журнал_дискавері_[компанія].md` — стан циклу
   (bootstrap з плану дискавері, reconcile з нових протоколів; розділ 8 — обіцяні замовнику артефакти A-N).
2. `session-client-prep-builder` — спершу перегонить робочі документи аналітика у Word
   (`питання_сесії-[N].docx` + `план_дискавері.docx`, якщо план оновився), потім — лист замовнику для підготовки:
   що підготувати, над чим подумати, кого запросити, короткий порядок денний. `.md` + брендований `.docx`
   (бланк ToDo вшито, ≤ 1 МБ). Вбудовані Humizer і фільтр чутливого.

### Блок «Аналіз сесії»

3. `session-transcript-analyzer` — ПІСЛЯ зустрічі розкладає транскрипт/нотатки сесії по реєстрах
   журналу (рішення D-N, виключення X-N, перенесення C-N, нові теми T-N, відкриті питання Q-N,
   обіцяні артефакти A-N) і проставляє статуси тем плану. **Головний writer журналу.** Дає короткий
   `аналіз_сесії-[N]` — вхід для `meeting-protocol-builder`. Коли всі теми закриті — сигналізує,
   що цикл можна завершувати.

## Цикл зустрічей

```
план_дискавері (ІНІЦІАЦІЯ, скіл 12)
    ↓
┌─→ session-question-builder    → питання_сесії-[N]_[компанія].md + журнал_дискавері_[компанія].md
│   session-client-prep-builder → питання_сесії-[N].docx + план_дискавері.docx (аналітику) + підготовка_замовника_сесії-[N]_[компанія].md + .docx → замовнику
│   БА фіналізує порядок денний → ЗУСТРІЧ (запис + транскрипт)
│   session-transcript-analyzer → оновлений журнал + аналіз_сесії-[N]
│   meeting-protocol-builder (ІНІЦІАЦІЯ, скіл 11) → протокол замовнику на погодження
│   обіцяні шаблони (опція) → витяг з протоколу + генерація Excel-форм + журнал розділ 8 (A-N)
│   перегляд плану дискавері (опція) → сигнали з журналу → оновити план чи лишити
└── ще зустрічі? Так → наступна ітерація (журнал уже оновив аналіз сесії) · Ні (всі теми ✅) → завершення дискавері
```

Аналіз транскрипту сесії (головний writer журналу) — реалізовано. Блок завершення дискавері
(маппінг scope, gap-аналіз, Discovery-звіт, міст у ТР) — окремий плагін `odoo19-discovery-closeout` (у розробці).

## Збірка та інсталяція

```bash
cd "СКІЛИ_ДИСКАВЕРІ" && bash ЗІБРАТИ_plugin.sh
```

Інсталяційна картка з кнопкою з'являється лише коли `.plugin` доставлено через папку **outputs** сесії Cowork.

## QA

- `валідатор_скілів_дискавері.py` — інваріанти вмісту обох пакетів
  (`python3 валідатор_скілів_дискавері.py skills "../СКІЛИ_виправлені/skills"`).
- `docs/ТЕСТ_циклу_3-ітерації.md` — рантайм-тест пам'яті циклу після інсталяції.

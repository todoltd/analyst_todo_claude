# Інструкція: Підготовка до Установчої Зустрічі (Claude)

**Для кого:** BA, менеджер проекту, будь-хто хто готує зустріч з клієнтом  
**Коли використовувати:** після того як зібрали пакет підготовки (`підготовка_УЗ_[назва_компанії].md`) — до Установчої Зустрічі. Якщо `підготовка_УЗ` ще немає — спочатку запусти **блок 1: Аналіз інформації + дослідження клієнта**  
**Результат:** порядок денний, опитувальник з питаннями, гайд фасилітатора і документ для клієнта за 60–110 хвилин

---

## Що тут є

4 скіли які перетворюють аналітичний пакет на практичні матеріали для зустрічі:

```
підготовка_УЗ_[назва_компанії].md  (з блоку "Аналіз інформації")
        ↓
1. kickoff-agenda-finalizer   → agenda_[назва_компанії].md
        ↓
2. kickoff-question-builder   → питання_УЗ_[назва_компанії].md   (деталізує блоки agenda в питання)
        ↓
3. facilitation-guide-builder → facilitation_[назва_компанії].md  (сценарій спирається на питання)
        ↑
        (паралельно або після)
4. discovery-charter-writer   → charter_[назва_компанії].md
```

**Мінімальний маршрут:** кроки 1 + 2 (порядок денний + опитувальник)  
**Рекомендований маршрут:** кроки 1 + 2 + 3 (додатково гайд фасилітатора)  
**Повний маршрут:** всі 4 кроки (додатково документ для клієнта)

---

## Крок 1 — kickoff-agenda-finalizer

**Що робить:** створює два порядки денних — зовнішній (для клієнта) і внутрішній (для команди з нотатками)  
**Коли запускати:** перший — одразу після готовності `підготовка_УЗ`  
**Час:** 10–20 хвилин

### Як запустити

Напиши у Claude одне з:
- "підготуй порядок денний для Установчої Зустрічі [назва компанії]"
- "зроби agenda для Установчої Зустрічі"
- "склади план зустрічі з [назва компанії]"

### Що потрібно мати

Обов'язково:
- `підготовка_УЗ_[назва_компанії].md` (з блоку "Аналіз інформації + дослідження клієнта")

Бажано:
- `бриф_[назва_компанії].md` — якщо потрібні деталі стейкхолдерів

### Що отримаєш

Файл `agenda_[назва_компанії].md` з:
- **Зовнішня версія** — надсилається клієнту за 1–2 дні до зустрічі
- **Внутрішня версія** — для команди: питання, нотатки фасилітатора, тригери
- Адаптації під конкретного клієнта (скептик, обіцянки, відсутній досвід дискавері тощо)

### На що звернути увагу

- Зовнішню версію надсилай клієнту без внутрішніх нотаток
- Якщо є 🔴 ризик-обіцянка — скіл додав блок "Узгодження очікувань": не прибирай його

---

## Крок 2 — kickoff-question-builder

**Що робить:** розгортає блоки порядку денного в детальний опитувальник — конкретні питання з пріоритетами, полем для відповіді і прив'язкою кожного питання до входу плану дискавері  
**Коли запускати:** після `agenda` — деталізує її змістовні блоки  
**Час:** 10–15 хвилин

### Як запустити

Напиши:
- "склади список питань для Установчої Зустрічі [назва компанії]"
- "підготуй опитувальник для зустрічі"
- "які питання задати клієнту [назва компанії]"

### Що потрібно мати

Обов'язково:
- `agenda_[назва_компанії].md` (з кроку 1)

Бажано:
- `підготовка_УЗ_[назва_компанії].md` — реєстр невідомого і ризики як джерело питань

### Що отримаєш

Файл `питання_УЗ_[назва_компанії].md` з:
- Питання згруповані за темами (бізнес-контекст, scope, процеси, інтеграції, стейкхолдери тощо)
- Кожне питання: пріоритет (🔴 Критично / 🟡 Важливо / ⚪ Опційно), поле "Відповідь:", тег входу плану (`[scope]`, `[процеси]`, `[дані]`…)
- **Карта покриття плану** — таблиця, яка показує, чи вистачить зібраної інформації для побудови плану, і де прогалини
- **Парковка для дискавері** — глибокі питання, які НЕ задаємо на 60-хвилинній зустрічі

### На що звернути увагу

- Документ тільки для команди — не надсилати клієнту (поля "Навіщо" і теги внутрішні)
- Тримай поруч під час зустрічі і фіксуй відповіді прямо в полях — заповнений файл стане входом для блоку 3
- Не зачитуй підряд як анкету: це орієнтир, а живий сценарій дає гайд фасилітатора (крок 3)
- Поважай рамки: 45 хв змістовної частини, фокус — scope; без цифр бюджету і без підрахунку користувачів

---

## Крок 3 — facilitation-guide-builder

**Що робить:** покроковий сценарій ведення зустрічі — репліки, переходи, маркери відповідей, нестандартні ситуації  
**Коли запускати:** після `agenda` (і бажано після `питання_УЗ` — тоді сценарій вплете готові питання)  
**Час:** 15–25 хвилин

### Як запустити

Напиши:
- "підготуй гайд фасилітатора для Установчої Зустрічі [назва компанії]"
- "зроби сценарій ведення зустрічі"
- "гайд фасилітатора для [назва компанії]"

### Що потрібно мати

Обов'язково:
- `agenda_[назва_компанії].md` (з кроку 1)

Бажано:
- `питання_УЗ_[назва_компанії].md` — щоб вплести детальні питання у сценарій
- `підготовка_УЗ_[назва_компанії].md` — для ризиків і стейкхолдерів

### Що отримаєш

Файл `facilitation_[назва_компанії].md` з:
- Профіль кожного учасника від клієнта з тактикою
- Сценарій по кожному блоку: відкриваюча фраза, питання, маркери відповідей
- 9 сценаріїв нестандартних ситуацій (скептик, scope-розмова, затягування, виходить ключовий учасник тощо)
- Чеклист після зустрічі

### На що звернути увагу

- Документ тільки для команди — не надсилати клієнту
- Маркери відповідей (✅ / ⚠️) — прив'язані до реалій цього клієнта, читай уважно
- Поле "Нотатник" у розділі "⚡ Перед зустріччю" — призначити хто записує до початку зустрічі

---

## Крок 4 — discovery-charter-writer

**Що робить:** короткий документ (~1 сторінка) для клієнта: що таке дискавері, що робимо, що отримаєте, що потрібно від вас  
**Коли запускати:** паралельно з кроками 2–3 або після — не залежить від них  
**Час:** 10–15 хвилин

### Як запустити

Напиши:
- "підготуй хартію дискавері для [назва компанії]"
- "зроби документ для клієнта про дискавері"
- "поясни клієнту що таке дискавері [назва компанії]"

### Що потрібно мати

Мінімум:
- Назва компанії клієнта і мета проекту (1 речення)

Бажано:
- `підготовка_УЗ_[назва_компанії].md` або `бриф_[назва_компанії].md` — для персоналізації

### Що отримаєш

Файл `charter_[назва_компанії].md` — документ для клієнта:
- Пояснення дискавері простою мовою
- Персоналізована мета для цього конкретного проекту
- Що ми будемо робити і скільки часу займе
- Deliverables по завершенні
- Що потрібно від клієнта (час, доступ, контакт)
- Відповіді на часті запитання

### Коли надсилати клієнту

- **Ідеально:** за 1–2 дні до Установчої Зустрічі разом з зовнішньою версією agenda
- **Прийнятно:** роздати на початку Установчої Зустрічі
- Не надсилай після зустрічі — документ призначений щоб зняти тривогу до неї

### Коли можна пропустити

- Клієнт з досвідом дискавері або ERP-впровадження — вже розуміє процес
- Коротке engagement без повноцінного дискавері

---

## Швидкий старт (мінімальний маршрут)

Якщо часу мало — два кроки:

```
1. kickoff-agenda-finalizer  → agenda_[назва_компанії].md
2. kickoff-question-builder  → питання_УЗ_[назва_компанії].md
```

Займе 20–30 хвилин. Зовнішню версію agenda надіслати клієнту; по внутрішній + опитувальнику вести зустріч.

---

## Типові сценарії

### Сценарій А: є повний пакет, є час
→ Запускай всі 4 кроки по черзі. Займе 60–75 хвилин.

### Сценарій Б: Установча Зустріч завтра, часу мало
→ Кроки 1 + 2 (agenda + опитувальник). Займе 25–35 хвилин. Опитувальник гарантує, що зберемо все для плану.

### Сценарій В: клієнт без досвіду з дискавері / перше впровадження ERP
→ Кроки 1 + 2 + 4 (agenda + опитувальник + charter). Хартія знімає тривогу і формує правильні очікування.

### Сценарій Г: новий BA на Установчій Зустрічі або складний клієнт (скептик, обіцянки)
→ Кроки 1 + 2 + 3 (agenda + опитувальник + гайд фасилітатора). Гайд дає впевненість і готові відповіді, опитувальник — повноту.

### Сценарій Д: повторний клієнт (новий проект)
→ Кроки 1 + 2 — клієнт вже знає процес, потрібен адаптований порядок денний і свіжий опитувальник по новому scope.

## Часті запитання

**Чим опитувальник (`питання_УЗ`) відрізняється від питань у agenda?**  
В `agenda` питання згруповані по блоках зустрічі і служать структурі. `питання_УЗ` — це робочий інструмент BA: кожне питання має пріоритет, поле для відповіді і тег входу плану дискавері, а "Карта покриття плану" показує, чи зберемо все потрібне. Опитувальник можна тримати відкритим під час зустрічі і фіксувати відповіді прямо в ньому.

**Чи можна запустити question-builder або facilitation-guide без готової agenda?**  
Технічно так (вони підхоплять `підготовка_УЗ`), але результат буде загальним. Обидва адаптуються під структуру конкретного порядку денного — без нього вийде шаблонно.

**Хартія (charter) — обов'язкова?**  
Ні. Вона найкорисніша для клієнтів без досвіду з дискавері або ERP. Для досвідчених клієнтів — можна пропустити.

**Зовнішня і внутрішня agenda — в одному файлі?**  
Так, обидві версії в одному `agenda_[назва_компанії].md`. Зовнішню версію скопіюй і надішли клієнту окремо — не пересилай весь файл.

**Що якщо `підготовка_УЗ` не було зроблено?**  
Скіли працюють і з брифом. Результат менш точний але корисний. В agenda, опитувальнику і гайді фасилітатора будуть позначки де даних не вистачає.

---

## Що далі після цього блоку

Після Установчої Зустрічі — запускай **блок 3: Аналіз Установчої Зустрічі + план дискавері**.
Заповнений на зустрічі `питання_УЗ` бери з собою у блок 3 — це структуроване джерело відповідей.

```
[ УСТАНОВЧА ЗУСТРІЧ ]
        ↓ транскрипт / нотатки + заповнений питання_УЗ
Блок 3: Аналіз Установчої Зустрічі + план дискавері
  → kickoff-transcript-analyzer  → аналіз_Установчої Зустрічі_[назва_компанії].md
  → discovery-plan-builder       → план_дискавері_[назва_компанії].md
        ↓
[ СТАРТ ДИСКАВЕРІ ]
```

Інструкція по блоку 3: `3. Аналіз Установчої Зустрічі + план дискавері / ІНСТРУКЦІЯ.md`

# odoo19-discovery-initiation

Методологія етапу **ІНІЦІАЦІЯ** для проєктів впровадження Odoo 19: 12 скілів, що ведуть BA та менеджера
проєкту від передачі клієнта продажником до готового плану дискавері.

## Скіли (3 блоки)

**Блок 1 — Аналіз інформації + дослідження клієнта**
1. `sales-handover-analyzer` — бриф з матеріалів продажів
2. `client-web-researcher` — дослідження клієнта в інтернеті
3. `competitor-mapper` — конкурентне середовище (опційно)
4. `industry-briefing` — галузевий контекст (опційно)
5. `discovery-context-builder` — зведений пакет підготовки до УЗ

**Блок 2 — Підготовка до Установчої Зустрічі**
6. `kickoff-agenda-finalizer` — порядок денний (зовнішній + внутрішній)
7. `kickoff-question-builder` — опитувальник з прив'язкою до входів плану
8. `facilitation-guide-builder` — сценарій ведення зустрічі
9. `discovery-charter-writer` — хартія дискавері для клієнта (опційно)

**Блок 3 — Аналіз Установчої Зустрічі + план дискавері**
10. `kickoff-transcript-analyzer` — аналіз транскрипту/нотаток
11. `meeting-protocol-builder` — **універсальний Протокол обстеження** для клієнта (.md + .docx, крок Humizer; калібровано на 41 реальному протоколі)
12. `discovery-plan-builder` — план дискавері

## Збірка та інсталяція

```bash
cd "СКІЛИ_виправлені" && bash ЗІБРАТИ_plugin.sh
```
Версія береться з `.claude-plugin/plugin.json`. Інсталяційна картка з кнопкою з'являється,
коли `.plugin` доставлено через папку **outputs** сесії Cowork.

## Залежність

Фірмовий бланк **вшито в скіл**: `skills/meeting-protocol-builder/assets/protocol_template.docx`
(оптимізований, ≤1 МБ). Файл `Шаблон_Протокол_обстеження.docx` у робочій папці — джерело бланку та резерв.

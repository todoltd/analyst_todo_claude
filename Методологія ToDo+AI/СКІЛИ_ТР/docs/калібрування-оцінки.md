# Калібрування оцінки (Обсяг робіт)

Орієнтири трудозатрат за **реальними ТР ToDo**. Це не тариф, а база для калібрування —
завжди звіряй з конкретикою задачі і давай діапазон. Версія 0.1, seed на 3 ТР; уточнюй у міру
накопичення даних (див. розділ «Як уточнювати»).

| Фільтри каталогу (15206) | 3 | 2,75 | 6 | 3,5 | **15,25** | 2 прості задачі: логіка видимості фільтрів + toggle «вибрані товари» |

## Друковані форми (QWeb) — дані RSE (формат ЧЧ:ХХ)
Проект RSE веде оцінку у форматі **години:хвилини** (не десятковому). Реальні дані по друкованих формах:

| ТР (запит) | ТР | Архіт. | Розробка | Тест | Інстр. | Всього | Суть |
| :-- | :-: | :-: | :-: | :-: | :-: | :-: | :-- |
| Проформа MWM (13992) | 4:00 | 0:15 | 7:00 | 0:30 | 0:15 | **12:00** | QWeb-проформа, 2 шаблони за умовою (повна/нульова), двомовна |
| Комерційний інвойс MWM (14505) | 2:00 | — | 5:00 | 0:30 | 0:15 | **~7:45** | QWeb-інвойс, лише EN, нові поля в partner/move |
| Комплектування до виробництва (13888) | 2:45 | — | 6:15 | 1:30 | — | **10:30** | кастомна QWeb-форма stock.picking, підміна через boolean |

**Орієнтир: QWeb друкована форма ≈ 7–12 год.** Складність ↑ від мультимовності, 2 шаблонів за умовою,
нових полів у `res.partner`/`account.move`, попередження про незаповнені поля.
> Формат часу — за проектом: Elfa/Олімпіус — десятковий (`2,75`), RSE — `ЧЧ:ХХ` (`2:45`). Зберігай той,
> що вже у документі/проекті.

## Похідні правила

**Аналітика (ТР + Архітектура разом):** ~3–6 год, слабо залежить від розміру задачі
(у вибірці: 5,75 / 4,5 / 2,75 год). Орієнтир: ~3 год для дрібної доробки, ~5–6 год для великої.

**Тестування:** ~30–50% від годин Розробки (у вибірці: 0,58 / 0,30 / 0,29). Для дрібних
доробок частка вища. Додавай фіксований оверхед, якщо «потрібно розгортати локальне середовище».

**Розробка — за архетипами (год):**

| Архетип доробки | Орієнтир, год |
| :-- | :-: |
| Зміна відображення (поля канбану, прибрати артикул, перейменувати поле) | 1–3 |
| Нове поле related/computed + групування/пошук | 1–3 |
| Domain на поле / простий фільтр у поданні | 2–4 |
| Фільтр за новим атрибутом/категорією | ~4 |
| Кнопка-дія з логікою (створення запису, оновлення, зміна стадії) + AC | 3–6 |
| Новий тип контакту + compute `display_name` + record rule + search | ~7 |
| Великий інтерактивний функціонал (інлайн-ввід, запис у модель по полях) | ~15–25 |
| Нова сутність/модуль (документ + рядки + стадії + права + smart-кнопки) | десятки (рахуй по складових) |

## Метод оцінки
1. Розклади задачу на архетипи Розробки (вище), просумуй → `Розробка`.
2. `Тестування` = 30–50% від Розробки (+ оверхед на локальне середовище за потреби).
3. `Технічне рішення` + `Архітектура` = аналітичний оверхед 3–6 год (за розміром).
4. `Всього` = сума; **перевір арифметику**.
5. Дай **діапазон** (мін–макс), познач допущення, від яких залежить оцінка.

**Приклад звірки:** доробка «фільтр e-com + 2 зміни відображення» ≈ Розробка 4 + 2×2 = 8 год;
Тест ~3 год; аналітика ~4 год → **~15 год (13–17)**. Збігається з порядком ТР 15206.

## Межі та чесність
- Оцінка — це судження, не факт. Не давай одну точну цифру там, де доречний діапазон.
- Великі/невідомі задачі (нова сутність, складна синхронізація, міграція) — закладай буфер
  і явно познач ризики, що можуть зсунути оцінку.
- Якщо AC ще не написані — оцінка груба; спершу `tr-usecases-acceptance`, кількість і складність
  AC прямо корелюють з годинами Розробки і Тестування.

## Як уточнювати
У міру закриття ТР додавай рядки у таблицю «Реальні дані» з фактичними годинами з «Обсяг робіт»
завершених документів (статус «Виконано» в реєстрі). Коли вибірка зросте — перерахуй частки
(тест/розробка, аналітичний оверхед) і діапазони архетипів. Реальні години беруться з полів
«Обсяг робіт» ТР, а не з календарних дат старту/фінішу реєстру.

> **Стан даних (станом на повний перегляд корпусу).** Точки калібрування: 3 з Elfa (доробки) + 3 з RSE
> (друковані форми QWeb, формат ЧЧ:ХХ). Корпуси Олімпіус (≈40 ТР), Bixter (HR) і ДЕМЗ переглянуто
> повністю — там **«Обсяг робіт» практично скрізь порожній**, тож звідти годинних даних не додалося.
> Коли почнуть заповнювати години — додавай їх сюди (особливо цінні виробничі/інтеграційні/HR-архетипи).

# tr-registry-update — реєстр «Перелік технічних рішень»

## Мета
Тримати реєстр актуальним: кожне ТР залоговане з ID задачі, посиланням, датами і статусом.
Реєстр — точка правди по життєвому циклу доопрацювань, тож запис має бути узгоджений з полями ТР.

## Інструменти та середовище
**Реєстр:** Google Sheet «Перелік технічних рішень» (Drive) → `search_files` / `read_file_content`.
**Вхід:** `ТР_[назва].md` (звідки беремо Назву, Номер запиту, посилання) та/або дані від користувача.
**Важливе обмеження:** прямий запис рядків у наявний Google Sheet через конектор недоступний —
скіл **готує рядок** і віддає його користувачу для вставки (або інструктує, що оновити). Видаляти/
перезаписувати в Drive не можна.

## Структура реєстру (стовпці)
```
| Назва доопрацювання | ID задачі | Посилання на ТР або задачу | Документ | Дата старту | Статус | Дата закінчення |
```
**Статуси (життєвий цикл):** `Аналіз та підготовка рішення` → `Погодження клієнта` →
`Виконано` / `Скасовано`. Посилання на задачу: `https://erp.todo.ltd/odoo/action-388/<...>/tasks/<ID>`.

## Крок 0: Визнач вхідні дані
- Знайди `ТР_*.md` (або візьми дані з чату): Назва доопрацювання, ID задачі (= «Номер запиту»),
  посилання на задачу/документ.
- Прочитай поточний реєстр (`read_file_content`), щоб:
  - перевірити, чи вже є рядок з цим **ID задачі** (тоді це **оновлення**, не новий рядок);
  - звірити формат і набір статусів, що реально використовуються.
- Якщо незрозуміло, яку дію робимо (новий запис vs зміна статусу) — спитай.

## Крок 1: Сформуй дію
**Новий запис:** заповни всі стовпці. «Дата старту» — сьогодні або дата початку аналізу;
«Статус» — зазвичай `Аналіз та підготовка рішення`; «Дата закінчення» — порожньо.
**Оновлення статусу:** познач новий статус і, якщо `Виконано`/`Скасовано`, простав «Дата закінчення».
Звір статус із реальним станом ТР (напр. `Погодження клієнта` — після `tr-review`; `Виконано` — коли
розробка і тестування завершені).

## Крок 2: Віддай результат
Покажи готовий рядок у вигляді, зручному для вставки в таблицю, наприклад:
```
Назва доопрацювання | ID задачі | Посилання | Документ | Дата старту | Статус | Дата закінчення
умови відображення фільтрів | 15206 | https://erp.todo.ltd/.../tasks/15206 | ТР_фільтри.md | 19.03.2026 | Погодження клієнта |
```
Дай чітку інструкцію: «Додай цей рядок у Google Sheet "Перелік технічних рішень"» (для нового)
або «Онови рядок з ID 15206: Статус → Виконано, Дата закінчення → [дата]» (для зміни).

## Pre-save верифікація
- [ ] ID задачі збігається з «Номер запиту» в ТР; перевірено на дубль у реєстрі;
- [ ] Статус — з дозволеного набору і відповідає реальному стану ТР;
- [ ] Для `Виконано`/`Скасовано` проставлена «Дата закінчення»; дати у форматі `ДД.ММ.РРРР`;
- [ ] Посилання валідне; для нового запису заповнені всі стовпці.

## Наступний крок
Повідом: «Рядок реєстру готовий — встав/онови в "Перелік технічних рішень". [Якщо Погодження клієнта]
Далі експорт у брендовий .docx для клієнта?»

## Чого НЕ робити
- Не вигадуй ID задачі і дати; бери з ТР/ERP або питай.
- Не став статус `Виконано`, поки розробка/тестування фактично не завершені.
- Не намагайся писати в Sheet «в обхід» — готуй рядок для ручної вставки.

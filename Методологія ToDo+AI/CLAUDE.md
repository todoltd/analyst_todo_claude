# Контекст папки: Методологія ToDo+AI (етапи ІНІЦІАЦІЯ + ДИСКАВЕРІ)

Робоча папка методології впровадження Odoo 19 — два плагіни: `odoo19-discovery-initiation`
(ІНІЦІАЦІЯ, 12 скілів) і `odoo19-discovery-sessions` (ДИСКАВЕРІ — цикл зустрічей, блок «Підготовка питань», 2 скіли).

## Канонічні артефакти

- `СКІЛИ_виправлені/` — **джерело істини пакета ІНІЦІАЦІЯ**: 12 скілів + `.claude-plugin/plugin.json` (версія тут) + `docs/` (інструкції) + `README.md` + `ЗІБРАТИ_plugin.sh` (авто-версійна збірка).
- `СКІЛИ_ДИСКАВЕРІ/` — **джерело істини пакета ДИСКАВЕРІ**: 2 скіли (`session-question-builder` — опитувальник сесії + журнал циклу, контракт журналу в `references/journal_template.md`; `session-client-prep-builder` — лист замовнику, бланк у `assets/letter_template.docx`) + `docs/` (майстер-інструкція, інструкція блоку, `ТЕСТ_циклу_3-ітерації.md`) + `валідатор_скілів_дискавері.py` (перевіряє ОБИДВА пакети) + власний `ЗІБРАТИ_plugin.sh`. План розробки: `ПЛАН_розробки_ДИСКАВЕРІ_блок_Підготовка-питань.md`.
- `odoo19-discovery-initiation-X.Y.Z.plugin`, `odoo19-discovery-sessions-X.Y.Z.plugin` — зібрані інсталяційні файли (тримати лише актуальні).
- `bpmn_odoo19_discovery_СТАТУС-скілів.html` — **канонічна BPMN-діаграма** (копія `bpmn_odoo19_discovery.html` у папці «…Дискавері» — тримати ідентичною їй).
- `ІНІЦІАЦІЯ_Майстер-інструкція.docx` — друкована майстер-інструкція (12 скілів, діаграма вшита).
- `Шаблон_Протокол_обстеження.docx` — джерело фірмового бланку (оптимізована копія вшита в скіл `meeting-protocol-builder/assets/`; бланк листа підготовки похідний від нього — в `session-client-prep-builder/assets/`; цей файл — резерв).
- `діаграма_процесу_ІНІЦІАЦІЯ.png` — рендер канонічної діаграми (⚠️ станом на 04.06.2026 відстає від HTML — перегенерувати при нагоді).

## Аналітика та історія

- `ПЛАН_аналізу_протоколів_EDELWEIS.md`, `АНАЛІЗ_протоколів_EDELWEIS_каталог.md` — аналіз 41 реального протоколу (база калібрування скіла протоколу).
- `ІСТОРІЯ_створення_скілів_ІНІЦІАЦІЯ.md` — журнал змін методології (вести далі, обидва етапи).
- `Архів/` — історичні QA-матеріали (звіти перевірок 5×5, плани, валідатор ІНІЦІАЦІЇ).

## Правила роботи

1. Скіли редагувати в `СКІЛИ_виправлені/skills/` (ІНІЦІАЦІЯ) або `СКІЛИ_ДИСКАВЕРІ/skills/` (ДИСКАВЕРІ), потім підняти версію у відповідному `plugin.json` → `bash ЗІБРАТИ_plugin.sh` того пакета.
2. **Інсталяція:** картка з кнопкою встановлення з'являється лише коли `.plugin` доставлено через папку **outputs** сесії Cowork (не з цієї папки напряму).
3. Авторська копія скілів/інструкцій — у сусідній папці «Методологія ToDo+AI Дискавері» (дерева `1. ІНІЦІАЦІЯ` і `2. ДИСКАВЕРІ`) — при змінах синхронізувати свідомо в обидва боки.
4. При зміні процесу оновлювати разом: скіли → інструкції (docs/ + nested) → BPMN (канон + копії) → майстер-інструкцію .docx → ІСТОРІЮ.
5. Контракт журналу циклу (`журнал_дискавері_…`) — спільний для майбутніх блоків ДИСКАВЕРІ: майбутній скіл аналізу транскриптів сесій стане основним writer'ом журналу; зміни контракту — лише через `references/journal_template.md` + ІСТОРІЮ.
